台北捷運運量 × AI 洞察分析儀表板 — 功能說明書
1. 專案概述
本專案旨在解決交通資料分析師在處理大量捷運運量數據時，面臨的手動整理繁瑣、視覺化製作耗時以及數據解讀主觀等痛點。透過整合前端開發技術與 Gemini AI，本工具可實現「上傳即分析」，在 10 秒內生成互動圖表與專業級的數據洞察報告。
2. 核心功能模組
2.1 資料導入模組
多格式上傳：支援拖放（Drag and Drop）或點擊上傳 CSV 檔案。
自動解析：整合 Papa Parse 函式庫，自動將 CSV 文字轉換為結構化 JSON 資料。
編碼相容性：特別優化處理台灣政府公開資料常見的 utf-8-sig (BOM) 編碼問題，避免亂碼。
欄位自動對齊：支援中文（日期、站名、進站人次）與英文（date, station, entries）欄位標籤辨識。
2.2 數據統計與摘要模組
系統會自動從原始資料中提取並計算以下關鍵指標：
關鍵指標卡 (Stats Bar)：
總進站人次（自動轉換為「百萬」單位）。
統計車站總數。
最熱門車站名稱及其運量。
資料統計涵蓋月份數。
排行運算：自動篩選出運量前 10 名的車站。
2.3 互動式視覺化模組
使用 Chart.js 渲染動態圖表，提供以下視覺化呈現：
月度趨勢折線圖：針對運量前五大的車站，繪製其各月份的消長趨勢，支援滑鼠懸停顯示詳細數值。
各站總運量排名圖：以水平長條圖呈現前十大熱門車站的累積運量對比，並以漸層色區分排名。
2.4 Gemini AI 洞察分析模組
本專案最核心的加值功能，將統計摘要傳送至 Gemini API 進行自動化解讀：
自動分析範疇：
整體趨勢：判斷運量成長、下降或穩定。
熱點站別解析：結合地理位置或商業特性分析熱門站點成因。
異常點偵測：指出運量異常飆升或下滑的月份/車站。
具體建議：針對交通局提供 2-3 條具體的政策或營運建議。
生成速度：於資料上傳後 10 秒內完成專業級繁體中文報告。
3. 系統架構與技術棧
技術層面	使用工具/技術	說明
前端架構	HTML5, CSS3, Vanilla JavaScript	輕量化架構，無需複雜安裝即可運行
資料解析	Papa Parse	處理大型 CSV 並支援 BOM 頭處理
圖表渲染	Chart.js	負責互動式折線圖與長條圖製作
人工智能	Gemini 3 Flash API	負責自然語言處理與數據洞察生成
資料傳遞策略	前端摘要傳遞 (Method C)	僅傳遞統計結果而非原始數千行資料，節省 Token 並加速回應
4. 使用者流程 (User Workflow)
進入儀表板：使用者開啟 HTML 檔案。
上傳資料：將從「台北市政府資料開放平台」下載的捷運運量 CSV 拖入上傳區。
瀏覽圖表：系統即時顯示統計數字與互動圖表。
獲取 AI 洞察：系統自動觸發 Gemini API，在頁面下方生成深度的文字分析報告。
匯出結果（選配）：使用者可透過瀏覽器列印功能或截圖保存分析報告。
5. 錯誤處理機制
檔案格式檢查：若上傳非 CSV 檔案，系統將跳出警示訊息。
解析失敗提示：若 CSV 格式不符（缺少必要欄位），將告知使用者正確的欄位格式要求。
API 金鑰防護：若 API Key 失效或網路不通，AI 洞察區域將顯示錯誤代碼並建議檢查網路連線。
6. 未來擴展方向
跨年度對比：支援同時上傳多個檔案，進行年度同比 (YoY) 分析。
地圖整合：整合 Leaflet.js，將運量數據直接呈現在台北捷運路網地圖上。
深度分析模型：針對大型資料集改用 Gemini File API 進行更深層的長文本分析。
